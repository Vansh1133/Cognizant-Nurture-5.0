# Week 3 - TDD using JUnit 5, Mockito & SLF4J

## Overview

This repository contains the mandatory hands-on exercises completed during Week 3 of the Java FSE training. The focus of this week was Test-Driven Development (TDD), unit testing using JUnit 5, mocking with Mockito, and logging using SLF4J.

---

## Technologies Used

- Java 17
- Maven
- JUnit 5
- Mockito
- SLF4J
- Logback
- IntelliJ IDEA

---

## Mandatory Hands-ons

### 1. JUnit Basic Testing

#### Exercise 1 - Setting Up JUnit
- Created a Maven project
- Configured JUnit 5 dependencies
- Wrote and executed basic unit tests

#### Exercise 3 - Assertions in JUnit
- Used different assertion methods:
    - assertEquals()
    - assertTrue()
    - assertFalse()
    - assertNull()
    - assertNotNull()

#### Exercise 4 - Arrange-Act-Assert (AAA) Pattern
- Implemented AAA testing pattern
- Used @BeforeEach for test setup
- Created clean and maintainable unit tests

---

### 2. Mockito

#### Exercise 1 - Mocking and Stubbing
- Created mock objects
- Used when() and thenReturn()
- Tested service methods without real dependencies

#### Exercise 2 - Verifying Interactions
- Verified method calls using verify()
- Tested repository interaction
- Ensured expected method invocation

---

### 3. SLF4J Logging

#### Exercise 1 - Logging Error Messages and Warning Levels
- Configured SLF4J with Logback
- Implemented logging using Logger and LoggerFactory
- Logged INFO, WARN and ERROR messages
